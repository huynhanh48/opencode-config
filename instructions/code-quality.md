# Code Quality

- Optimize for clean, professional, maintainable code.
- Remove needless boilerplate when language features or standard helpers make the code simpler.
- Prefer straightforward expressions, early returns, and small cohesive helpers over deep nesting.
- Reduce line count only when readability is preserved or improved.
- Avoid code golf, obscure operator tricks, or dense one-liners that reduce maintainability.
- Keep behavior explicit when shortening code would hide important edge cases or error handling.
